# Баг-репорты Robo Clicker

Дефекты подготовлены по результатам статического анализа исходников и изолированной проверки DOM/основных сценариев. Перед закрытием или передачей разработчику рекомендуется воспроизвести их в целевом HTTP-окружении и приложить screenshot/video/console/network evidence.

## BUG-001 — Выбранный язык не восстанавливается после перезагрузки

| Поле | Значение |
|---|---|
| Модуль | Настройки / Локализация |
| Severity | **Major** |
| Priority | **High** |
| Статус | Подтвержден статическим анализом |
| Окружение | Все браузеры; локальный режим и SDK-режим |

**Предусловие**  
Браузер с локалью ru-RU; игра запущена по HTTP.

**Шаги**  
1. Открыть Настройки.  
2. Выбрать EN.  
3. Перезагрузить страницу.

**Фактический результат**  
После запуска applyStartupLanguage снова выбирает язык платформы/браузера и перезаписывает сохранённый выбор.

**Ожидаемый результат**  
При наличии language в localStorage использовать сохранённый пользователем язык.

**Evidence / основание**  
LANGUAGE_KEY записывается в applyLanguage, но не читается в applyStartupLanguage.

**Источник в проекте:** `script.js: applyStartupLanguage / LANGUAGE_KEY`

## BUG-002 — Кнопка «ПОЕХАЛИ!» в окне «Об игре» не запускает игру

| Поле | Значение |
|---|---|
| Модуль | Главное меню / Об игре |
| Severity | **Minor** |
| Priority | **Medium** |
| Статус | Подтвержден статическим анализом |
| Окружение | Все браузеры |

**Предусловие**  
Открыто главное меню.

**Шаги**  
1. Нажать «Об игре».  
2. Нажать «ПОЕХАЛИ!».

**Фактический результат**  
Модалка закрывается, вызывается goToMenu(), пользователь остаётся в главном меню.

**Ожидаемый результат**  
Кнопка с призывом «ПОЕХАЛИ!» запускает игру либо имеет другую однозначную подпись.

**Evidence / основание**  
Обработчик about-go-btn вызывает goToMenu(), а не startGame().

**Источник в проекте:** `script.js: 1865–1871`

## BUG-003 — Звуки сначала запрашиваются из отсутствующей папки sounds

| Поле | Значение |
|---|---|
| Модуль | Мультимедиа / Ресурсы |
| Severity | **Minor** |
| Priority | **Medium** |
| Статус | Подтвержден статическим анализом |
| Окружение | HTTP-сервер |

**Предусловие**  
Проект развёрнут без дополнительной папки sounds.

**Шаги**  
1. Открыть DevTools Network.  
2. Запустить игру.  
3. Нажать игровые кнопки.

**Фактический результат**  
Первичные пути sounds/click.wav, sounds/start.wav и sounds/menu.wav отсутствуют; используется fallback на корневые файлы.

**Ожидаемый результат**  
Основные src сразу указывают на существующие файлы, без 404 и задержки fallback.

**Evidence / основание**  
В репозитории *.wav лежат в корне, createGameSound сначала использует sounds/*.wav.

**Источник в проекте:** `script.js: sounds; структура проекта`

## BUG-004 — Глобальный keydown блокирует клавиши Shift и I

| Поле | Значение |
|---|---|
| Модуль | Клавиатура / Accessibility |
| Severity | **Major** |
| Priority | **High** |
| Статус | Подтвержден статическим анализом |
| Окружение | Все desktop-браузеры |

**Предусловие**  
Страница в фокусе.

**Шаги**  
1. Нажать Shift.  
2. Нажать клавишу I.  
3. Проверить event.defaultPrevented и работу assistive/браузерных сценариев.

**Фактический результат**  
Для keyCode 16 и 73 всегда вызываются preventDefault и stopPropagation.

**Ожидаемый результат**  
Блокируются только документированные опасные комбинации; обычные клавиши работают.

**Evidence / основание**  
Глобальный document.addEventListener("keydown") содержит e.keyCode==16 || e.keyCode==73.

**Источник в проекте:** `script.js: конец файла`

## BUG-005 — Основной игровой объект недоступен с клавиатуры

| Поле | Значение |
|---|---|
| Модуль | Игровой экран / Accessibility |
| Severity | **Major** |
| Priority | **High** |
| Статус | Подтвержден статическим анализом |
| Окружение | Все браузеры; screen reader/keyboard-only |

**Предусловие**  
Игра запущена; используется только клавиатура.

**Шаги**  
1. Нажимать Tab до игрового объекта.  
2. Попытаться активировать Enter/Space.

**Фактический результат**  
#click-object — div без tabindex, role и keyboard handler; фокус на него не попадает.

**Ожидаемый результат**  
Главное действие игры доступно клавиатурой и имеет accessible name.

**Evidence / основание**  
index.html содержит <div id="click-object">; логика подписана только на pointerdown.

**Источник в проекте:** `index.html + script.js`

## BUG-006 — Кнопки закрытия «Об игре» и «Настройки» не имеют доступного имени

| Поле | Значение |
|---|---|
| Модуль | Модальные окна / Accessibility |
| Severity | **Minor** |
| Priority | **Medium** |
| Статус | Подтвержден статическим анализом |
| Окружение | Screen reader |

**Предусловие**  
Открыто окно «Об игре» или «Настройки»; включён screen reader.

**Шаги**  
1. Перейти Tab на кнопку ✕.  
2. Прослушать объявление screen reader.

**Фактический результат**  
Кнопки содержат только символ ✕ и не имеют aria-label/title с назначением.

**Ожидаемый результат**  
Например, «Закрыть окно Об игре» и «Закрыть настройки».

**Evidence / основание**  
close-about и close-settings не включены в ariaMap и не имеют aria-label в HTML.

**Источник в проекте:** `index.html: close-about/close-settings`

## BUG-007 — Условия достижений, связанных со скинами, считаются по количеству owned

| Поле | Значение |
|---|---|
| Модуль | Достижения / Скины |
| Severity | **Major** |
| Priority | **High** |
| Статус | Подтвержден статическим анализом |
| Окружение | Все браузеры |

**Предусловие**  
Есть доступ к нескольким скинам.

**Шаги**  
1. Купить любые 5 скинов, не выполняя последовательность редких.  
2. Проверить «Огненный гардероб».  
3. Купить любые 4 скина и проверить «Император стиля».

**Фактический результат**  
Тип skinsEquipped возвращает ownedSkinIds.size; условие rare/consecutive не отслеживается. Для «все ultra» также проверяется только число.

**Ожидаемый результат**  
Отслеживать фактические редкости, equip-события и последовательность согласно описанию.

**Evidence / основание**  
getAchievementProgressValue: skinsEquipped → ownedSkinsWithoutDefault; achievement 57 goal=4 type=skins.

**Источник в проекте:** `script.js: achievements 56–59, getAchievementProgressValue`

## BUG-008 — Достижение «Все типы бустов» учитывает только три категории

| Поле | Значение |
|---|---|
| Модуль | Достижения / Бусты |
| Severity | **Major** |
| Priority | **High** |
| Статус | Подтвержден статическим анализом |
| Окружение | Все браузеры |

**Предусловие**  
Использован по одному бусту из Temporary, Permanent и Super.

**Шаги**  
1. Использовать три буста из разных категорий.  
2. Не использовать остальные типы.  
3. Проверить прогресс достижения «Император бустов».

**Фактический результат**  
boostTypesUsed хранит boost.category; после трёх категорий размер Set равен goal=3.

**Ожидаемый результат**  
Условие «все типы» должно учитывать идентификаторы всех 12 бустов либо формулировка должна говорить «все категории».

**Evidence / основание**  
boostTypesUsed.add(boost.category); achievement 68 goal=3.

**Источник в проекте:** `script.js: buyBoost/grantBoostForReward, achievement 68`
